# Visão do Produto — Escopo e Roadmap

## 1. O Fluxo Principal (Jornada do Usuário - V1)
O fluxo principal, projetado para o Mínimo Produto Viável (MVP), foca em agilidade e navegação intuitiva:
1. **Onboarding:** O usuário cria uma conta (E-mail/Senha) e é imediatamente redirecionado para a tela de Login.
2. **Autenticação:** Ao logar, acessa a tela de Início (Dashboard).
3. **Navegação:** Através de um menu lateral (hamburguer), o usuário acessa as grandes categorias de conhecimento (ex: *Prompts de IA, Comandos Git, Queries SQL, Backend, Frontend*).
4. **Interação:** Ao clicar em uma categoria, a lista de documentos é exibida. O usuário seleciona um item para abrir seus detalhes.
5. **Ação Rápida:** Na visualização do documento, existe um botão de "Copiar" (para resgate imediato) e opções para Editar ou Excluir (CRUD).

> **Sugestão Inclusa na V1 (Baseado em UX):** Para maximizar a agilidade que você tanto preza, incluiremos uma **Barra de Busca Global** logo na tela inicial, permitindo achar um prompt pelo nome sem precisar abrir o menu. Também teremos uma lista de "Favoritos" para os itens que você usa todo dia!

## 2. Roadmap de Evolução (Macro)

O detalhamento completo do roadmap por versão (V1/V2/V3) é responsabilidade única do documento `docs/08-roadmap/01-roadmap-detalhado.md`, para evitar duas fontes divergentes. Duas ideias de produto para versões futuras, capturadas aqui durante a fase de Visão do Produto, já estão incorporadas lá:
- **Modo Portfólio** (V2): tela pública listando os melhores guias/prompts, como um currículo vivo das habilidades técnicas do usuário.
- **Assistente de IA embutido e auto-classificação** (V2): IA nativa para tirar dúvidas, melhorar prompts/queries e sugerir automaticamente a categoria de um conteúdo colado.
